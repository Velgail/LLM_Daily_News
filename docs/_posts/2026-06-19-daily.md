### 1. ChatGPT に「スケジュールタスク専用ページ」登場──自動化タスクを一元管理、Pulse は廃止

> 「あとでやっておいて」をChatGPTが字義通り実行する時代へ。OpenAIが2年越しの自動化機能をついに本格UIに昇格させた。

OpenAIは6月17日、ChatGPTにスケジュールタスクの専用管理ページ「Scheduled」を追加した。Web・iOS・Android の各プラットフォームで、Go・Plus・Pro・Business・Enterprise 各プランのユーザーが利用可能。ユーザーはサイドバーから現在稼働中のタスクを一覧確認し、一時停止・編集・削除ができるほか、「毎朝株価をまとめて送って」「月曜にミーティング準備の要約を作って」といった時刻または時間帯指定のプロンプトを設定できる。

特筆すべきは、従来提供していた「Pulse」（パーソナライズされた日次ブリーフィング機能）がこのリリースとともに廃止されたことだ。Pulseの機能はスケジュールタスクシステムへ統合され、ユーザーは自分の好みに合わせた自動ブリーフィングを自由に設定できるようになる。また、ChatGPTがウェブや接続アプリを定期的にチェックし、異常を知らせる「監視タスク」機能も提供される。信頼性改善のために backend を刷新し、粒度の細かい通知と webhook サポートも追加された。プランごとの同時タスク上限は Go が3件、Plus が5件、Business が10件、Pro/Enterprise は最大15件。

> OpenAI と Anthropic が AWS 上で存在感を固める中、Musk 率いる xAI もついに Bedrock エコシステムへ参入した。

6月15〜16日、AWSはxAIのGrok 4.3をAmazon Bedrock上で正式提供開始したと発表した。Grok 4.3はBedrock初の xAI モデルとなり、企業ユーザーは既存のAWS環境からそのままAPIを呼び出せる。最大の特徴は可変推論レベル（none/low/medium/high）の設定機能で、タスクの重みに応じてコストと精度をトレードオフできる。コンテキストウィンドウは100万トークンで、カスタマーサポート・ウェブ開発・判例調査・財務書類Q&Aといった企業向けワークロードに特化した最適化が施されている。

推論エンジンには Amazon Bedrock 向けに新設された「Mantle」が採用されており、ツール呼び出し・構造化出力・ストリーミング応答をサポート。xAIにとってこの統合は、x.ai API・X.com・iOS/Androidアプリに続く第4の流通経路となり、競合するOpenAIの aws.amazon.com/openai-on-aws（4月29日発表）やAnthropicのClaude Platform on AWS（5月17日）と同じフィールドに立つことを意味する。AWS上での「フロンティアモデル競争」がいよいよ本格化した。

### 3. 智谱AI「GLM-5.2」がMITライセンスで公開──100万トークンコンテキスト・744Bパラメータ・GPT-5.5比6分の1コスト

> オープンウェイトでGPT-5.5を複数ベンチマークで超えるモデルが登場。中国AI勢の「コスト破壊」が再び加速する。

中国AI企業・智谱AI（Z.ai）は6月13日、GLM-5.2をリリースした。744Bの総パラメータを持つMixture-of-Expertsモデルで、推論時の活性化パラメータは約40B。最大の見どころは100万トークンのコンテキストウィンドウで、前バージョン（GLM-5.1）の20万トークンから5倍拡張した。出力上限も13万1,072トークンと大幅に増加している。

VentureBeatの分析によれば、GLM-5.2はGPT-5.5に対して長期水平線のコーディングベンチマーク複数で上回るスコアを記録しながら、APIコストはGPT-5.5の約6分の1。MITライセンスのオープンウェイトで地域制限なしに公開されている点も重要で、中国・米国問わず開発者が自由に利用・改変できる。コンテキスト長の増加に伴う計算効率を維持するため「IndexShare」と呼ばれるアーキテクチャ最適化を採用。エージェント向けのコーディングタスクや長時間ソフトウェアエンジニアリングを主要ユースケースとして設計されており、独自の「デュアル思考努力モード」（高・最大）も搭載している。

### 4. Google「Gemini-SQL2」が自然言語→SQL変換で初めて80%超え──BIRDベンチマークで世界最高精度を達成

> データベースに聞くだけで SQL を書いてくれる未来が、本当に産業レベルの精度になりつつある。

GoogleResearchは6月12日、テキスト→SQL変換システム「Gemini-SQL2」を発表した。Gemini 3.1 Proに専門的な事後学習とスキャフォールディングを施した特化システムで、BIRD（Business Recommended Dataset）シングルモデルリーダーボードにおいて80.04%の実行精度を記録。BIRDは37の専門ドメインにわたる95データベース・12,751問を対象に、生成されたSQLが実際に正しく実行されるかを評価するベンチマークで、従来の最高スコアは競合モデルが軒並み70〜73%台にとどまっていた。OpenAIのGPT-5.5-xhigh（72.8%）とAnthropicのClaude Opus 4.6（70.9%）を大きく引き離す。

核心技術は「スキーマリンカー」──数百テーブルを持つ複雑なDBから最も関連性の高いテーブルとカラムを自動抽出し、Gemini 3.1 Proの長コンテキスト能力を使ってスキーマ定義やビジネス用語集全体を一括処理する。現在のところGemini-SQL2はパブリックAPIとして提供されていないが、GoogleはBigQuery・Looker等の自社データサービスに統合していく方針を示しており、企業のデータアクセス民主化が一気に加速する可能性がある。なお人間のエキスパートスコアは92.96%で、まだ12.9ポイントの差が残る。
