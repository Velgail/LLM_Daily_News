Cloudflare は 2026 年 5 月、自社のグローバルネットワーク上で LLM を稼働させるために独自に開発した推論エンジン「Infire」の詳細を公開した。Rust で記述されたこのエンジンは、GPU のメモリ・ネットワーク I/O・計算効率を最大化するよう設計されており、非搭載の H100 NVL GPU 上で vLLM 0.10.0 と比較して最大 7% 高速な推論を達成、スループット（トークン毎秒）は最大 20% 向上した。また、入力処理と出力生成を別々の最適化されたシステムに分離する「 disaggregation」アーキテクチャを採用し、GPU を効率的に管理するカスタム推論エンジンとして機能する。特筆すべきは、Kimi K2.5 のような超大型モデルでさえ 20 秒以内に起動できる点で、エッジ環境での LLM 展開における「コールドスタート問題」を実質的に解消しつつある。パイプライン並列・テンソル並列・エキスパート並列など複数の並列化方式をサポートし、低スペックハードウェアでの大型モデル実行も可能にする。

### 技術的インプリケーション

**Granite 4.1 の示す「dense vs. MoE」論争の決着**：MoE (Mixture-of-Experts) モデルは推論時のアクティブパラメータを絞ることで効率化を図るが、「ファインチューニングの複雑さ」「エキスパートルーティングの不安定性」「新しいタスクへの転移の難しさ」という課題を抱えてきた。IBM の結果は、良質なデータと多段階の学習プロセス（事前学習→教師あり微調整→強化学習）を組み合わせれば、dense モデルで十分に競争力を維持できることを証明した。エンタープライズ開発者にとって「ファインチューニングしやすいモデル」は実装コストの直接削減に繋がる。

**Cloudflare Infire の「推論の民主化」**：Infire が示すのは、推論インフラが「専用ハードを大量所有する者の独占物」から「エッジネットワーク上で効率的に分散できるコモディティ」へと移行しつつあるという方向性だ。vLLM 比 7% の速度向上は数字だけ見れば地味だが、「超大型モデルが 20 秒以内に起動できる」という副次効果は、サーバーレス LLM API の実用化に直結する。コールドスタートが解消されれば、使用量ゼロ時にインスタンスをゼロに落とし、リクエスト到来時に即起動するという、完全な従量課金モデルが成立する。
