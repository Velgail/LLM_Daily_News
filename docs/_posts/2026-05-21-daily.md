### 3. Google I/O 2026：「WebMCP」提案でWebがAIエージェント対応に——Chrome 149でオリジントライアル開始、ブラウザ内エージェントが構造化ツールを直接呼び出せる新標準

> Webサイトがみずから「このボタンを押して」「このフォームに入力して」とAIエージェントに対してAPIのように指示できる新Web標準「WebMCP」が登場。スクレイピング頼みの不確実な自動操作から、開発者が制御できる確実なエージェント連携へと一変させる可能性を秘める。

GoogleはI/O 2026のChrome開発者セッションで、「WebMCP（Web Model Context Protocol）」を新しいWeb標準として提案した。開発者がJavaScript関数やHTMLフォームを「エージェントが呼び出せるツール」として明示的に公開できる仕組みで、W3C Web Machine Learning Community GroupでMicrosoftとの共同でインキュベーション中の仕様だ。Chrome 149のオリジントライアルとして実験的提供が始まる。

従来のAIエージェントによるブラウザ自動操作はDOMのビジュアルスクレイピング頼みで、サイト変更のたびに壊れやすかった。WebMCPでは、命令型API（JavaScript関数定義）と宣言型API（HTMLフォームへのアノテーション）の2方式でエージェントが呼び出すべきツールをサイト側が明示できる。エージェントは旅行APIへのクエリや金融データ送信といった複雑なバックエンド操作を、機械精度で・ユーザー承認のもとで実行できるようになる。

この他にGoogle I/O 2026では15件のChrome向けAI更新が発表された。目玉は6月下旬のAndroidへのGemini in Chrome展開で、記事要約・カレンダー/Gmail連携・マルチステップタスクを自動処理する「Auto Browse」が提供される。また「Nano Banana」機能でAndroidのChromeからオンデバイス画像生成・編集が可能になる。
