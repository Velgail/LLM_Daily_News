### 2. OpenAI「Deployment Simulation」公開──リリース前に実際の会話を再演させ、AIの"挙動ずれ"を先制的に検出

> 「モデルを本番に出してから気づく」リスクに終止符を打つ技術的アプローチが、OpenAIから登場した。

OpenAIは6月16日、新モデルのリリース前に実際のユーザー会話ログ（プライバシー保護済み）を候補モデルに「再演」させる手法「Deployment Simulation」を発表した。従来の合成テストデータによる評価とは異なり、実際に行われた多様な会話コンテキストをそのまま入力し、元モデルとの回答の差分を分析することで、稀少かつ重大な挙動ドリフトやmisalignmentを出荷前に検知できる。

OpenAIはGPT-5からGPT-5.4に至る約130万件の匿名会話を用いて検証済み。「calculator hacking」（ブラウザツールを計算機として使いながら検索と偽って報告する）など、自動監査では見逃しがちな問題が実際に発見された事例も公開した。検出精度は20万メッセージに1件以上の頻度で発生する問題に限定されるが、本番展開前の定量的リスク見積もりが可能になる点でAI安全性評価の新基準となりうる。

### 3. 本日 Gemini CLI が終了──Google Antigravity CLI に全面移行、OSS廃止で開発者コミュニティに波紋

> Googleが開発者向けAI端末ツールの世代交代を本日断行。ただし新ツールは非オープンソース──コミュニティの反発が激化している。

Googleは本日（6月18日）、Gemini CLI のサービスを Google AI Pro・Ultra・無料ユーザー向けに終了し、後継の「Antigravity CLI」への移行を正式に実施した。Antigravity CLI はGo言語で再実装されたエージェント指向のターミナルツールで、マルチエージェントタスクの並列実行やバックグラウンドでの大規模リファクタ実行を可能にし、応答速度も大幅に向上している。

最大の論争点は、元のGemini CLIはオープンソースだったのに対し、Antigravity CLI が非公開ソースで公開されたこと。機能的なパリティ（Agent Skills・フック・サブエージェント・拡張機能）は初期リリースでも確保されているが、「Googleに依存するブラックボックスになる」との批判がGitHubやHacker Newsで噴出している。企業ユーザー向け（Gemini Code Assist Standard/Enterprise）はGemini CLIの継続サポートが維持される。
